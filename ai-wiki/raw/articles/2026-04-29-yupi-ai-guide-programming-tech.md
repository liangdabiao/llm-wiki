# 鱼皮的 AI 指南 - 4、AI 编程技术

最重要的部分：程序员不仅会用 AI 工具、用 AI 开发项目，还要能自主开发 AI 项目，把 AI 能力接入自己的项目。

> 金句：AI 时代，所有的传统业务都值得利用 AI 重塑一遍。

现在很多公司都在招能够开发 AI 项目的程序员，这是我们的机会。

---

## 一、AI 开发框架

| 场景 | 推荐框架 | 优势 |
|------|---------|------|
| Java 企业应用 | Spring AI | 无缝集成 Spring 生态 |
| 智能体开发 | LangChain4j | 完整 Agent 工具链 |
| 复杂工作流 | LangGraph | 可视化编排 |

### 1. Spring AI

- 作用：快速对接大模型、保存会话上下文、对接向量数据库实现 RAG
- 优势：更容易和主流 Java 开发框架 Spring 集成，上手难度低
- 官网：https://docs.spring.io/spring-ai/reference/getting-started.html

### 2. LangChain4j

- 作用：提供很多现成方法提高 AI 应用开发效率
- 优势：更灵活，更适合开发复杂的智能体
- 示例：智能文档分析系统 - 自动读取文档 → 调用搜索引擎补充背景知识 → 结合生成分析报告
- 官网：https://docs.langchain4j.dev/intro

**学习建议**：两个都要学，先从 Spring AI 学起，再学 LangChain4j 会更简单。

### 3. LangGraph

- 用图的结构组织和管理 AI 相关工作流
- 适合构建有状态、多代理的企业级 AI 大项目
- 比喻：多个 AI 智能体（分别负责生成文字、生成图片、组合文字和图片），LangGraph 是负责人，安排工作顺序，一起搞大事
- 官网：https://www.langchain.com/langgraph

## 二、AI 集成

开发 AI 应用的前提是要有大模型。大模型消耗算力，两种获取方式：

### 方案 1：AI 云服务

其他企业部署 AI 大模型，通过 API 接口提供使用，按量计费。

**推荐平台**：阿里云百炼、火山引擎、硅基流动、OpenAI 等。

**程序员重点掌握**：
1. 如何通过 API 接入云服务
2. 如何使用 AI 云服务创建智能体和配置参数
3. 如何选择合适的云服务（关注计费模式和服务质量）
4. 如何更低成本、更稳定使用云服务（学习 Prompt 工程和高可用技术）

### 方案 2：本地部署大模型

**为什么需要**：
- 很多企业刚需：数据无需上传至云端
- 保障数据安全性和隐私性
- 尤其医疗、金融等对数据安全极为敏感的行业

**实现方式**：使用 Ollama 工具，一键部署各种主流开源模型。

官网：https://ollama.com/

**现实问题**：部署难度不在于技术，而在于算力成本！

## 三、AI 领域业务

企业 AI 业务开发不止 AI 对话，还要掌握更复杂的业务开发：

### 1. RAG 知识库

**用途**：构建公司自己的问答系统或客服，基于企业真实数据作答，更准确贴合实际。

**工作流程**：
1. 通过文本嵌入模型，将企业各种文档转化为向量
2. 存入向量数据库
3. 用户提问时，系统在向量数据库检索相关向量数据
4. 找到最相似文档片段
5. 和问题一起输入大模型处理

**学习重点（也是面试重点）**：
- 主流向量数据库：Milvus、PGVector
- 文档抽取 / 转换 / 加载
- 索引构建
- 查询策略优化

### 2. 多模态

融合文本、图像、音频、视频等多种不同类型的数据模态，提高产品易用性，做更多创意功能。

**例子**：智能导购系统，支持文字描述、图片识别、语音理解购物需求。

**需要学习**：
- 模态转换技术：文本转语音（TTS）、语音转文本（STT）、光学字符识别（OCR）
- 都是现成工具库或云服务，掌握调用方法即可
- Spring AI、LangChain 等框架可调用不同模态的大模型

### 3. MCP 服务

Model Context Protocol，模型上下文协议。

可以理解为：提供给 AI 的各种服务，AI 利用这些服务实现更强大的功能。

**需要掌握**：
1. 如何在项目中接入别人的 MCP 服务，增强项目能力
2. 如何开发自己的 MCP 服务，让别人使用

**开发工具**：
- Spring AI 等开发框架
- MCPify：http://mcpify.ai/ 高手做的网站，一句话创建自己的 MCP 服务

### 4. ReAct 智能体

构建智能体的开发范式，目的是打造能够依据推理结果自主采取行动的智能体。

**开发涉及**：任务规划、工具调用、交互 I/O、异常处理等知识。

**工具调用方式**：
- Function Call
- MCP

**功能**：天气查询、文件读写、网页运行、信息检索、终端命令执行等。

**开发示例**：开发视频网站，用户说"帮我开发一个 Dilidili 视频网站并部署上线"，智能体流程：
1. 深入理解任务内容
2. 推理梳理执行步骤
3. 明确需求、设计方案
4. 搭建框架、生成代码
5. 部署上线
6. 执行过程中遇到问题，询问意见，重新推理调整行动方案

## 四、AI 工具链

### 1. 低代码平台 - Dify

- 官网：https://dify.ai/
- 主流低代码 AI 开发平台
- 通过拖拉拽方式构建 AI 智能体
- 创建知识库并导入文档
- 搭建复杂工作流
- 不会写代码也能搞复杂 AI 应用

### 2. 工具库

- **Apache Tika**：功能强大的文件解析器，支持 PDF、Word、Excel、PowerPoint 等各种文档
- **Playwright**：模拟浏览器行为工具库，运行网页、抓取网页数据、自动化测试
- **JSON 解析库**：GSON、Kryo
- **HTML 解析库**：jsoup

学习成本低，要用时看文档即可。

### 3. 部署工具

**云服务优先选择**：大厂提供的云服务，优先于价格和稳定性。

**个人学习/快速上线平台**：
- **Vercel**：https://vercel.com/ 适合前端应用，自动构建、在线浏览、CDN 分发、免费域名
- **Sealos**：https://sealos.io/ 云原生应用管理平台，Kubernetes 集群管理，容器化部署环境，弹性伸缩
- **Railway**：https://railway.com/ 轻松部署 Docker 容器，无需操心服务器配置运维，自带自动化构建工具、环境管理

**必备技术**：Docker 容器化技术，像 APP 安装包，轻松分发和部署应用程序。
