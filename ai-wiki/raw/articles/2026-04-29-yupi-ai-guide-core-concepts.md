# 鱼皮的 AI 指南 - 1、AI 核心概念

通过实操 Dify AI 应用开发平台，把 AI 核心概念串联起来。

## 1. 大模型 (LLM)

大模型是 AI 的大脑，具有海量参数的人工智能模型，通过大规模预训练获得广泛知识和能力。

不同大模型的参数规模、处理能力、接受的对话长度不一样。

## 2. 温度 (Temperature)

控制模型输出的随机性的参数：
- 温度值越高，输出越随机多样
- 温度值越低，输出越确定保守

## 3. 提示词 (Prompt)

输入给 AI 的内容，用来引导模型生成特定内容或执行特定任务。

**提示词质量 = AI 输出准确度**

分为两种：
- 系统提示词：整体约束 AI 输出内容，需要提前设置
- 用户提示词：用户自主输入内容，随用随输

## 4. Token

大语言模型处理文本的基本单位，可能是单词或标点符号。

- 模型输入和输出都按 Token 计算
- Token 越多，成本越高，输出速度越慢
- 价格：一般 100 万 Token 几十块钱

Token 计算工具：https://tiktoken.aigc2d.com/

## 5. API Key

调用大模型 API 的秘钥，每次扣费都会记在这个 Key 上。

需要去大模型官方申请调用秘钥。

## 6. 模型蒸馏

将复杂大模型的知识转移到更小模型中的技术。

目的：在保持模型性能的同时，减小模型体积，降低推理成本。

例如：Gemini 标准版和 Lite 轻量版。

## 7. 多模态

多模态大模型能够同时理解和处理多种类型的信息：
- 文本
- 图像
- 音频
- 视频

可以实现：图生文、文生图、文生视频等更智能的应用。

## 8. 上下文信息

可以上传 PDF 文件等内容，交给 AI 去总结。

几年前 ChatPDF 就是做这个的，现在人人都能实现了。

## 9. RAG 检索增强生成

利用外部知识库给 AI 补充知识，解决大模型缺少某些信息的问题。

**工作流程**：
1. 创建知识库，上传知识文档
2. 对文本进行切分，设置分块规则
3. 利用 Embedding 嵌入技术，将文本转换为向量表示
4. 写入向量数据库
5. 用户提问时，将问题转换为向量
6. 从知识库中检索和问题相关的信息
7. 将检索到的信息和问题一起输入大模型处理
8. 得到更准确的回答

## 10. Embedding 嵌入

将文本、图像等数据转换为向量表示的技术。

是 RAG 的核心技术之一。

## 11. API 接口

AI 应用可以通过 API 接口，在自己的代码程序中通过网络请求来调用。

## 12. 智能体 (Agent)

能够感知环境、进行推理、制定计划、做出决策并自主采取行动来实现目标的 AI 系统。

可以给智能体提供工具：网页搜索、查询天气、调用数据库等，让智能体完成更复杂的任务。

建议智能体选用思考能力更强的推理模型。

## 13. 思维链 CoT

Chain of Thought，让模型在处理复杂问题时，详细介绍中间步骤和推理过程。

解决问题：模型直接给出答案缺乏逻辑性和可解释性。

作用：让整个推理过程更透明，人们能够理解模型是如何得出结论的。

## 14. ReAct

结合推理（Reason）和行动（Act）的 AI 开发范式。

流程：模型先思考问题 → 推理分析并提出行动计划 → 执行行动 → 基于结果进一步推理。

循环进行，使 AI 能够更有效地解决复杂问题。

思考过程是公开可见的，增强可解释性。

## 15. 智能体工作流 Agentic Workflow

通过规划和编排，让智能体自由搭配功能，自动化实现各种复杂的任务。

有点像可视化编程。

适用场景：单一智能体不能完成的任务，如自动生成 100 个短视频、自动做个游戏并发布上线。

## 16. MCP 模型上下文协议

Model Context Protocol，用于实现 AI 与外部工具或数据的标准化交互。

简单来说：利用 MCP 服务，可以更方便地给 AI 集成不同的工具和数据，增强 AI 应用功能。
